# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| Latest | Yes |

This package is pre-1.0 and only the latest published version receives security
fixes.

## Reporting a Vulnerability

To report a security vulnerability, please email
[spencer@beggs.codes](mailto:spencer@beggs.codes).

You can also use
[GitHub Security Advisories](https://github.com/spencerbeggs/yaml-effect/security/advisories/new)
to report vulnerabilities privately.

Please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

We will acknowledge receipt within 72 hours and provide updates as we
investigate. We appreciate responsible disclosure and will credit reporters in
release notes unless anonymity is requested.
